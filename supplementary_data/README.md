# Supplementary Data (S4)

This folder is the single labeled entry point to the machine-readable data underlying the
manuscript. It contains no new files: each entry below is a symlink to the canonical
location under `outputs/`, so the analysis scripts and the supplementary bundle always read
the same files.

| Bundle | Path | Contents |
|---|---|---|
| Analysis tables | [`tables/`](tables) | Aggregate result CSVs and machine-readable report JSONs (panel summary, GEE, cross-judge agreement, accuracy trade-off, paraphrase robustness, decode stability, multiplicity/heterogeneity/power, adjudication, sensitivity). |
| Figures | [`figures/`](figures) | All manuscript figures as SVG, PNG, and per-figure PDF. |
| Rubric scores | [`scores/`](scores) | Primary judge (GPT-5.4-nano), cross-judge (Claude Sonnet 5), correctness judge (GPT-5.4-mini), and decode-replicate rubric scores as JSONL. |
| Clinician review | [`doctor_review/`](doctor_review) | De-identified three-clinician rating sheets (judge-validation, adjudication, and positive-enriched sensitivity sets) and the hidden answer keys. One excluded auto-templated adjudication sheet is retained with documentation. |

## Not included here

- **Raw model-output records** (`outputs/predictions/*.jsonl`) are **not** redistributed:
  they embed source-case text from the underlying licensed datasets (Real-POCQi,
  HealthBench, MedRBench). They are available from the corresponding author on reasonable
  request. See the manuscript's Data and Code Availability section.
- **Run logs** (`outputs/logs/`) are execution artifacts and are not version-controlled.

## Reproducing the reports

Each report file is regenerated by its named script in `analysis/` (for example
`analysis/cross_judge_robustness.py` → `tables/crossjudge_agreement_report.json`); the full
mapping is given in the manuscript's Data and Code Availability section.
